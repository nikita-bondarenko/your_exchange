# E2E Тесты для главной страницы

## Описание тестов

Все тесты находятся в файле `e2e/homepage.spec.ts` и проверяют функциональность главной страницы приложения.

## Список тестов

### 1. Переключение режима через тоггл "Обмен криптовалют" - "Платежи за рубеж"
- Проверяет наличие тоггла переключения режимов
- Проверяет переключение между режимами "Обмен" и "Платежи за рубеж"
- Проверяет изменение текста кнопки "Начать обмен" / "Начать платеж"
- Проверяет отображение соответствующих списков описаний

### 2. Навигация кнопки "Начать обмен"
- Проверяет, что кнопка "Начать обмен" ведет на страницу `/exchange/type`
- Убеждается, что навигация работает корректно

### 3. Навигация кнопки "Начать платеж"
- Проверяет, что кнопка "Начать платеж" ведет на страницу `/transfer-abroad/type`
- Убеждается, что навигация работает корректно

### 4. Проверка загрузки аватара с сервера
- Проверяет наличие кнопки профиля с аватаром
- Проверяет загрузку изображения аватара с сервера
- Проверяет, что изображение успешно загружено (не ошибка)
- Проверяет навигацию на страницу профиля при клике

### 5. Проверка секции "Дополнительно" и функциональности всех кнопок
- Проверяет открытие/закрытие секции "Дополнительно"
- Проверяет кнопку "Профиль" и навигацию на `/profile`
- Проверяет кнопку "Нас часто спрашивают" и навигацию на `/faq`
- Проверяет кнопку "Соглашение" (если присутствует) и открытие внешней ссылки
- Проверяет кнопку "Политика AML" (если присутствует) и открытие внешней ссылки
- Проверяет закрытие секции при повторном клике

### 6. Проверка кнопки "Поддержка" и логов в консоли
- Проверяет наличие кнопки "Поддержка"
- Проверяет логи в консоли после клика
- В браузере (не в Telegram WebApp) проверяет наличие сообщения об ошибке "User ID is required" или сообщения о закрытии приложения
- Проверяет отсутствие критических ошибок в консоли

### 7. Комплексная проверка: переключение режима и навигация
- Комплексный тест, проверяющий переключение режима и соответствующую навигацию
- Проверяет полный цикл: режим обмена → навигация → режим переводов → навигация

## Запуск тестов

### Запуск всех тестов
```bash
npm run test:e2e
```

### Запуск с видимым браузером
```bash
npm run test:e2e:headed
```

### Интерактивный режим (рекомендуется для отладки)
```bash
npm run test:e2e:ui
```

### Режим отладки
```bash
npm run test:e2e:debug
```

## Примечания

- Тесты автоматически запускают dev сервер перед выполнением
- Некоторые тесты могут быть пропущены (skip), если соответствующие функции не включены в проекте
- Тесты учитывают, что в браузере (не в Telegram WebApp) некоторые функции могут работать иначе
- Тесты проверяют логи консоли для кнопки поддержки, так как в браузере она не закрывает приложение
