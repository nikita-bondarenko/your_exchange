# Инструкция по тестированию

## Установка зависимостей

Перед запуском тестов необходимо установить зависимости:

```bash
npm install
# или
pnpm install
```

## Unit тесты (Jest + React Testing Library)

### Запуск тестов

```bash
# Запустить все тесты
npm run test

# Запустить тесты в режиме watch
npm run test:watch

# Запустить тесты с покрытием кода
npm run test:coverage
```

### Структура unit тестов

- Тесты находятся в `src/**/__tests__/**/*.test.tsx`
- Утилиты для тестирования: `src/__tests__/utils/test-utils.tsx`
- Конфигурация Jest: `jest.config.js`
- Настройки Jest: `jest.setup.js`

### Примеры тестов

Unit тесты для главной страницы находятся в:
- `src/b__pages/home/ui/__tests__/HomePage.test.tsx`

## E2E тесты (Playwright)

### Установка браузеров Playwright

При первом запуске необходимо установить браузеры:

```bash
npx playwright install
```

### Запуск e2e тестов

```bash
# Запустить все e2e тесты
npm run test:e2e

# Запустить с UI интерфейсом
npm run test:e2e:ui

# Запустить в видимом режиме (headed)
npm run test:e2e:headed

# Запустить в режиме отладки
npm run test:e2e:debug
```

### Структура e2e тестов

- E2E тесты находятся в `e2e/**/*.spec.ts`
- Конфигурация Playwright: `playwright.config.ts`
- E2E тесты для главной страницы: `e2e/homepage.spec.ts`

### Примечания

- E2E тесты автоматически запускают dev сервер перед выполнением
- Для работы e2e тестов необходимо, чтобы приложение запускалось на `http://localhost:3000`
- В CI/CD окружении убедитесь, что переменная `CI=true` установлена

## Что тестируется

### Unit тесты главной страницы

- Отображение заголовка страницы
- Отображение подзаголовка
- Отображение кнопок (Начать обмен/платеж, Поддержка)
- Отображение списка описаний для разных режимов
- Отображение секции "Дополнительно"
- Обработка запросов в процессе

### E2E тесты главной страницы

- Отображение основных элементов страницы
- Навигация при клике на кнопки
- Адаптивность на разных размерах экрана
- Отсутствие критических ошибок в консоли
